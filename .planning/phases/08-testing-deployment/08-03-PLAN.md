  <action>
    Create `.github/workflows/ci.yml` with 6 jobs per the locked decision (lint, backend-tests, frontend-tests, e2e-tests, build, deploy).

    **Trigger:** on push to main, on pull_request to main.

    **Job 1: `lint`** (runs-on: ubuntu-latest)
    - Checkout code (actions/checkout@v4)
    - Setup Python 3.12 (actions/setup-python@v5) with pip cache
    - Setup Node 20 (actions/setup-node@v4)
    - Install ruff: `pip install ruff`
    - Lint backend: `ruff check services/ tests/`
    - Install and lint guest frontend: `cd frontend && npm ci && npm run lint`
    - Install and lint staff frontend: `cd frontend-staff && npm ci && npm run lint`

    **Job 2: `backend-tests`** (needs: lint)
    - Checkout code
    - Start infrastructure containers: `docker compose up -d auth_db room_db booking_db rabbitmq mailpit minio`
    - Wait for databases to be healthy: loop with `docker compose exec <db> pg_isready` and `sleep 10`
    - Setup Python 3.12 with pip cache
    - Install dependencies:
      ```
      pip install -r services/auth/requirements.txt
      pip install -r services/room/requirements.txt
      pip install -r services/booking/requirements.txt
      pip install -r services/gateway/requirements.txt
      pip install pytest pytest-asyncio httpx
      pip install shared/
      ```
    - Generate RSA keys for JWT: `mkdir -p keys && openssl genrsa -out keys/jwt-private.pem 2048 && openssl rsa -in keys/jwt-private.pem -pubout -out keys/jwt-public.pem`
    - Set environment variables for tests (DATABASE_URLs pointing to localhost:5433/5434/5435, JWT key paths, RABBITMQ_URL, MAIL_SERVER, MINIO vars)
    - Run tests: `pytest tests/ -x --tb=short`
    - Teardown (if: always): `docker compose down -v`

    **Environment variables for backend-tests step:**
    ```yaml
    env:
      AUTH_DATABASE_URL: postgresql+asyncpg://auth_user:auth_pass@localhost:5433/auth
      ROOM_DATABASE_URL: postgresql+asyncpg://room_user:room_pass@localhost:5434/rooms
      BOOKING_DATABASE_URL: postgresql+asyncpg://booking_user:booking_pass@localhost:5435/bookings
      JWT_PRIVATE_KEY_PATH: keys/jwt-private.pem
      JWT_PUBLIC_KEY_PATH: keys/jwt-public.pem
      RABBITMQ_URL: amqp://hotel:hotel_pass@localhost:5672/
      MAIL_SERVER: localhost
      MAIL_PORT: 1025
      MINIO_ENDPOINT: localhost:9000
      MINIO_ACCESS_KEY: minioadmin
      MINIO_SECRET_KEY: minioadmin
      MINIO_BUCKET: hotelbook
      MINIO_SECURE: "false"
      ROOM_SERVICE_URL: http://localhost:8002
    ```

    **Job 3: `frontend-tests`** (needs: lint)
    - Checkout code
    - Setup Node 20 with npm cache
    - Guest tests: `cd frontend && npm ci && npm test -- --run`
    - Staff tests: `cd frontend-staff && npm ci && npm test -- --run`

    **Job 4: `e2e-tests`** (needs: [backend-tests, frontend-tests])
    - Checkout code
    - Build and start full stack: `docker compose up -d --build`
    - Setup Node 20
    - Install Playwright: `cd e2e && npm ci && npx playwright install --with-deps chromium`
    - Wait for services: `timeout 120 bash -c 'until curl -f http://localhost:8000/health 2>/dev/null; do sleep 2; done'`
    - Run E2E: `cd e2e && npx playwright test`
    - Upload test artifacts on failure (actions/upload-artifact@v4): path e2e/test-results/
    - Teardown (if: always): `docker compose down -v`

    **Job 5: `build`** (needs: e2e-tests)
    - Checkout code
    - Build all Docker images to verify they compile successfully:
      ```
      docker compose -f docker-compose.yml -f docker-compose.prod.yml build
      ```
    - This validates that all Dockerfiles (backend services + frontend production builds) produce valid images before deploy.
    - Note: Images are not pushed to a registry -- the deploy step builds on EC2 directly. This job ensures the build does not break.

    **Job 6: `deploy`** (needs: build, if: github.ref == 'refs/heads/main' && github.event_name == 'push')
    - Uses appleboy/ssh-action@v1
    - with: host ${{ secrets.EC2_HOST }}, username ${{ secrets.EC2_USER }}, key ${{ secrets.EC2_SSH_KEY }}
    - script:
      ```
      cd /opt/hotelbook
      git pull origin main
      docker compose -f docker-compose.yml -f docker-compose.prod.yml up -d --build --remove-orphans
      docker system prune -f
      ```

    **Caching strategy:**
    - Python: actions/setup-python with cache: 'pip'
    - Node: actions/setup-node with cache: 'npm' and cache-dependency-path for each frontend
    - Docker: Docker layer cache via BuildKit (DOCKER_BUILDKIT=1 env var)
  </action>
