**CheckInDialog (`src/components/checkin/CheckInDialog.tsx`):**
- Uses shadcn Dialog component
- Props: `booking` (BookingResponse), `open` (boolean), `onOpenChange`, `onSuccess?`
- Title: "Check in {guest_first_name} {guest_last_name}"
- Fetch available rooms for booking's room_type_id using `useAvailableRooms(booking.room_type_id)`
- Auto-assign: Sort available rooms by `floor ASC` then `room_number ASC` (client-side sort since backend doesn't support sort param). Pick first room.
- Display auto-assigned room: "Room {room_number} ({floor}F)" with "Change" link
- If "Change" clicked: Show dropdown (shadcn Select) of all available rooms of that type, each labeled "Room {room_number} ({floor}F)"
- If no rooms available: Show inline error "No available rooms of this type. Please assign manually or change room type." in red
- Confirm button: "Check In to Room {room_number}" (accent `#0F766E`, white text)
- Cancel button: "Cancel" (outline variant)
- On confirm: Call `useCheckIn().mutateAsync({ bookingId: booking.id, roomId: selectedRoom.id })`
- On success: Close dialog, show toast "{guest_first_name} {guest_last_name} checked in to Room {room_number}" via sonner
- On error: Show toast "Something went wrong. Please try again."
- Loading state: Button shows spinner, disabled during mutation
- Dialog traps focus (handled by shadcn Dialog)
