# Phase 6 — UI Design Contract

> Visual and interaction contract for the Staff Dashboard. Generated by gsd-ui-researcher, verified by gsd-ui-checker. This is a SEPARATE frontend app (`frontend-staff/`) with a dark admin visual identity distinct from the guest site.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn (initialize with `npx shadcn@latest init` in `frontend-staff/`) |
| Preset | default (New York style — matches guest frontend for component consistency) |
| Component library | Radix UI (via shadcn/ui) |
| Icon library | lucide-react |
| Font | Inter (sans-serif) via Google Fonts CDN |

**shadcn components to install:** button, card, input, label, select, dialog, skeleton, separator, badge, dropdown-menu, avatar, form, table, pagination, sheet, tabs, sonner, popover, sidebar, tooltip

**Additional libraries:**
- sonner (toast notifications)
- date-fns (date manipulation -- already in stack)

**Shared patterns replicated from guest frontend (not imported):**
- API client structure (Axios with JWT interceptor)
- Zustand auth store pattern (with role-based checks)
- TanStack Query hook patterns
- StatusBadge component (same color mapping)

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding, table cell compact padding |
| sm | 8px | Compact element spacing, sidebar icon-to-label gap |
| md | 16px | Default element spacing, card padding, form field gaps |
| lg | 24px | Section padding, sidebar group spacing |
| xl | 32px | Layout gaps between major dashboard sections |
| 2xl | 48px | Page-level vertical padding |
| 3xl | 64px | Not used in staff dashboard (no hero sections) |

Exceptions: Touch targets minimum 44px height for all interactive elements. Sidebar nav items minimum 40px height for dense but clickable targets.

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px | 400 (regular) | 1.5 | Table cells, descriptions, form help text, card content |
| Label | 12px | 400 (regular) | 1.4 | Form labels, column headers, meta text, badge text, sidebar nav items, stat labels |
| Heading | 20px | 600 (semibold) | 1.2 | Page titles, section headings, card titles |
| Display | 32px | 600 (semibold) | 1.1 | Metric values on overview cards (today's check-ins count, occupancy %) |

Font stack: `"Inter", ui-sans-serif, system-ui, -apple-system, sans-serif`

**Note:** Staff dashboard uses denser typography than guest frontend (14px body vs 16px, 12px labels vs 14px) to fit more operational data on screen. Two weights: 400 (regular) for content and labels, 600 (semibold) for headings, display metrics, and emphasis.

## Layout Contract

### Responsive Breakpoints

| Breakpoint | Width | Layout Adaptation |
|------------|-------|-------------------|
| mobile | < 768px | Sidebar hidden (hamburger toggle), single column cards, stacked metrics |
| tablet | 768px - 1279px | Sidebar collapsed (icons only, 64px), two-column metric cards |
| desktop | >= 1280px | Sidebar expanded (240px), full dashboard layout, multi-column grids |

### Sidebar Navigation (fixed left)

- Width: 240px expanded, 64px collapsed (icons only), hidden on mobile
- Background: `#1E293B` (slate-800) with `border-right: 1px solid #334155`
- Top: Logo "HotelBook" in heading weight (600), teal accent, 16px. Below: "Staff" label in 12px slate-400
- Nav items: lucide icon (20px) + label (12px regular), 40px height, 8px left padding
- Active item: Left 3px border in accent color + `rgba(15,118,110,0.15)` background
- Hover: `#283548` background
- Nav groups with 12px uppercase slate-400 section headers
- Mobile: Hamburger icon in top-left triggers Sheet slide-out from left
- Collapse toggle: Chevron icon at sidebar bottom to collapse/expand on tablet. `aria-label="Collapse sidebar"` when expanded, `aria-label="Expand sidebar"` when collapsed. Tooltip on hover showing the same label.
- Collapsed sidebar nav items: Each icon-only nav item shows a tooltip on hover with the nav item label (e.g., "Overview", "Reservations"). Each icon has `aria-label` matching the nav item label.

**Navigation items:**
1. **Overview** (LayoutDashboard icon) - Today's dashboard
2. **Reservations** (CalendarCheck icon) - All bookings list with search/filter
3. **Check-in/out** (LogIn icon) - Today's arrivals and departures
4. **Room Status** (BedDouble icon) - Room status board
5. **Guests** (Users icon) - Guest profile lookup
6. **Reports** (BarChart3 icon) - Placeholder, disabled, tooltip "Coming in Phase 7"

**Bottom section:**
- Staff avatar + name + role badge
- Logout button (LogOut icon). `aria-label="Log out"`. Tooltip on hover showing "Log out".

### Top Bar (sticky, right of sidebar)

- Height: 56px
- Background: `#0F172A` (slate-900) with `border-bottom: 1px solid #334155`
- Left: Page title (heading size, 20px semibold)
- Right: Notification bell icon (placeholder, no functionality), staff avatar dropdown

### Overview Page (Home)

**Primary visual anchor: today's operational snapshot** -- four metric cards at the top give staff immediate situational awareness of the day's operations.

- Top row: 4 metric cards in a row (2x2 on tablet, 4x1 on desktop, stacked on mobile)
  - "Today's Check-ins" -- count of confirmed bookings with today's check-in date
  - "Today's Check-outs" -- count of checked-in bookings with today's check-out date
  - "Occupancy" -- percentage of occupied rooms, circular or bar indicator
  - "Rooms to Clean" -- count of rooms in cleaning status
- Each metric card: Icon (accent color) top-left, metric value (display 32px semibold), label below (12px slate-400)
- Card background: `#1E293B`, border: `1px solid #334155`, rounded-lg (8px)
- Below metrics: Two-column layout on desktop
  - Left: "Today's Arrivals" list (compact card rows with guest name, room type, confirmation #, "Check In" button)
  - Right: "Today's Departures" list (compact card rows with guest name, room #, "Check Out" button)
- Each list: Maximum 5 items shown, "View All" link at bottom

### Reservation List Page

- Top: Search bar (full width) + filter row (status dropdown, date range picker, room type filter)
- Search placeholder: "Search by guest name or confirmation number..."
- Below filters: Card grid layout
  - Desktop: 3 columns
  - Tablet: 2 columns
  - Mobile: 1 column
- Each reservation card:
  - Top: Guest name (heading weight) + status badge (right-aligned)
  - Middle: Confirmation # (monospace, slate-400), room type, check-in/out dates
  - Bottom: Quick action buttons (contextual based on status)
    - Confirmed: "Check In" (accent), "Cancel Booking" (destructive outline)
    - Checked In: "Check Out" (accent), "View" (outline)
    - Others: "View" (outline) only
  - Card: `#1E293B` background, `#334155` border, rounded-lg, 16px padding
  - Card hover: `#283548` background, transition 150ms
- Pagination: Numbered pages at bottom, 12 cards per page
  - Active page: accent background, white text
  - Inactive: slate-800 background, slate-300 text
  - Chevron arrows for prev/next

### Check-in/out Page

- Two tabs: "Arrivals" and "Departures"
- Each tab shows a filtered list of today's relevant bookings
- Arrivals list items:
  - Guest name, room type booked, confirmation #, special requests (if any)
  - "Check In" button (accent) on right
- Check-in flow (dialog):
  1. Dialog opens: "Check in {Guest Name}"
  2. Auto-assigned room displayed: "Room {number} ({floor}F)" with "Change" link
  3. If "Change" clicked: dropdown of all available rooms of that type
  4. Auto-assign algorithm: lowest floor first, then lowest room number (predictable, easy for housekeeping)
  5. Confirm button: "Check In to Room {number}"
  6. Success toast: "{Guest Name} checked in to Room {number}"
- Check-out flow (dialog):
  1. Dialog opens: "Check out {Guest Name} from Room {number}"
  2. Shows stay summary: dates, nights, total amount
  3. Confirm button: "Confirm Check-out"
  4. Success toast: "{Guest Name} checked out. Room {number} marked for cleaning."

### Room Status Board

- Grid of room cards showing all rooms
- Group by floor (collapsible floor sections)
- Each room card: Small card (120px x 80px approx)
  - Room number (heading weight, centered)
  - Status indicator (colored dot + label)
  - Room type (12px, truncated)
- Status colors on cards:
  - Available: green-600 dot + "Available" label
  - Occupied: teal-700 dot + "Occupied" label
  - Cleaning: amber-600 dot + "Cleaning" label
  - Maintenance: red-600 dot + "Maintenance" label
  - Inspected: blue-500 dot + "Inspected" label
- Click room card: Popover showing full room details + status change actions

### Guest Profile Page

- Search bar at top: "Search by guest name or email..."
- Search results: List of matching guests
- Selected guest profile:
  - Top: Guest name (heading), email, phone
  - Below: "Booking History" heading
  - Table: Confirmation #, room type, dates, status badge, total amount
  - Table sorted by check-in date descending (most recent first)
  - Read-only -- no edit capability

## Interaction States

### Buttons
- Default: Accent background (`#0F766E`), white text, rounded-md (6px)
- Hover: Lighten to `#0D9488` (teal-600), cursor pointer
- Active/pressed: Scale 0.98 transform
- Disabled: 30% opacity, cursor not-allowed
- Loading: Spinner icon replaces text, disabled state
- Outline variant: Transparent background, `#334155` border, slate-300 text. Hover: `#283548` background
- Destructive variant: Transparent background, red-600 text, red-600 border. Hover: `rgba(220,38,38,0.15)` background
- Ghost variant: No border, no background. Hover: `#283548` background. Used for sidebar nav items.

### Form Inputs
- Default: `#1E293B` background, `#334155` border, rounded-md (6px), slate-100 text, 14px body
- Placeholder: `#64748B` (slate-500)
- Focus: Accent border (`#0F766E`), ring shadow `0 0 0 2px rgba(15,118,110,0.3)`
- Error: Red-600 border, red error message below (12px label)
- Disabled: 20% opacity

### Cards
- Default: `#1E293B` background, `#334155` border, rounded-lg (8px)
- Hover: `#283548` background, transition 150ms ease
- No shadow (dark theme uses borders, not shadows, for elevation)

### Tables
- Header row: `#0F172A` background, 12px uppercase slate-400 text, sticky
- Body rows: `#1E293B` background, alternating `#172033` for even rows
- Row hover: `#283548` background
- Cell padding: 12px horizontal, 8px vertical
- Border: `1px solid #334155` between rows

### Page Transitions
- Route transitions: None (instant)
- Content loading: Skeleton placeholders with `animate-pulse`, `#1E293B` on `#0F172A` background
- Sheet/drawer (mobile sidebar): Slide in from left, 300ms ease-out, dark overlay
- Toast: Slide in from top-right, auto-dismiss after 5 seconds
- Dialog: Fade + scale in from center, 200ms ease

## Accessibility Contract

- All interactive elements keyboard-navigable (tab order follows visual order)
- Focus ring visible on all focusable elements (accent color ring on dark background)
- Color contrast minimum 4.5:1 for all text on dark backgrounds (slate-100 on slate-900 = 13.3:1, passes)
- Status badges use text labels (not color alone)
- Sidebar nav items have aria-current="page" for active item
- Dialog confirmations trap focus within dialog
- Table headers use `<th>` with scope="col"
- Metric cards use aria-label describing the metric: "Today's check-ins: {count}"
- Skip-to-content link at top of page (visually hidden until focused)
- Room status board cards have aria-label: "Room {number}, {status}, {room type}"
- Sidebar collapse toggle: `aria-label="Collapse sidebar"` when expanded, `aria-label="Expand sidebar"` when collapsed. Shows tooltip on hover.
- Logout button: `aria-label="Log out"`. Shows tooltip on hover.
- Collapsed sidebar nav items: Each icon-only nav item has `aria-label` matching its label text and shows a tooltip on hover displaying the label (e.g., "Overview", "Reservations", "Check-in/out", "Room Status", "Guests", "Reports").
