**Auth service changes:**

Add to `services/auth/app/services/user.py`:
```python
async def search_users(db: AsyncSession, query: str, skip: int = 0, limit: int = 20) -> tuple[list[User], int]:
    """Search users by first name, last name, or email using ILIKE."""
    search_term = f"%{query}%"
    filters = or_(
        User.first_name.ilike(search_term),
        User.last_name.ilike(search_term),
        User.email.ilike(search_term),
    )
    count_result = await db.execute(select(func.count()).select_from(User).where(filters))
    total = count_result.scalar()
    result = await db.execute(select(User).where(filters).offset(skip).limit(limit))
    users = list(result.scalars().all())
    return users, total
```
Import `or_` from sqlalchemy.

Add to `services/auth/app/api/v1/users.py` a new search endpoint:

@router.get("/search", response_model=UserListResponse)
async def search_users_endpoint(

    skip: int = 0,
    limit: int = 20,
    db: AsyncSession = Depends(get_db),
    _current_user: User = Depends(require_staff),

    """Search users by name or email (staff only)."""
    users, total = await search_users(db, q, skip=skip, limit=limit)

Endpoints (all forward auth header from request):

1. `POST /api/v1/staff/check-in/{booking_id}` -- Orchestrate check-in:
   - Parse `room_id` from request JSON body
   - Step 1: POST to `{BOOKING_SERVICE_URL}/api/v1/bookings/staff/{booking_id}/check-in` with `{"room_id": room_id}`
   - If booking response not 200, return that error
   - Step 2: POST to `{ROOM_SERVICE_URL}/api/v1/rooms/{room_id}/status` with `{"new_status": "occupied"}`
   - If room service fails, log warning but still return booking success (graceful degradation per project convention)
   - Return booking response

2. `POST /api/v1/staff/check-out/{booking_id}` -- Orchestrate check-out:
   - Step 1: POST to `{BOOKING_SERVICE_URL}/api/v1/bookings/staff/{booking_id}/check-out`
   - If booking response not 200, return error
   - Extract `room_id` from booking response JSON
   - Step 2: If room_id exists, POST to `{ROOM_SERVICE_URL}/api/v1/rooms/{room_id}/status` with `{"new_status": "cleaning"}`
   - Graceful degradation if room service fails
   - Return booking response

3. `GET /api/v1/staff/overview` -- Aggregate overview:
   - Parallel fetch: today bookings from booking service + room status board from room service
   - From today bookings: count arrivals and departures
   - From room board: count total rooms, occupied rooms (occupancy %), rooms in cleaning status
   - Return JSON: `{"arrivals_count": N, "departures_count": N, "occupancy_percent": N, "rooms_to_clean": N, "arrivals": [...], "departures": [...]}`
   - Use `asyncio.gather` for parallel requests

4. `GET /api/v1/staff/guests/search?q=name` -- Forward to auth service search:
   - GET `{AUTH_SERVICE_URL}/api/v1/users/search?q={q}` with auth header
   - Return pass-through response

5. `GET /api/v1/staff/guests/{user_id}/profile` -- Aggregate guest profile:
   - Parallel fetch: user detail from auth service + bookings by user from booking service
   - GET `{AUTH_SERVICE_URL}/api/v1/users/{user_id}`
   - GET `{BOOKING_SERVICE_URL}/api/v1/bookings/staff/by-user/{user_id}`
   - Merge into: `{"user": {...}, "bookings": {...}}`
   - Graceful degradation if booking service fails

Use `httpx.AsyncClient(timeout=15.0)` for all calls. Import `asyncio` for parallel calls. Follow the same pattern as `services/gateway/app/api/booking.py` (pass-through Response objects, json import for serialization).
