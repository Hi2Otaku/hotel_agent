**1. Booking service aggregation service** (`services/booking/app/services/reports.py`):

Create three async functions:

- `get_revenue_by_room_type(db: AsyncSession, start_date: date, end_date: date, group_by: str) -> list[dict]`
  - Query: `SELECT room_type_id, date_trunc(group_by, check_in) as period, SUM(total_price) as revenue, COUNT(*) as count FROM bookings WHERE check_in >= start_date AND check_in < end_date AND status IN ('confirmed', 'checked_in', 'checked_out') GROUP BY room_type_id, period ORDER BY period`
  - `group_by` parameter: "day" for ranges < 30 days, "week" for 30-90 days, "month" for > 90 days
  - Return `[{"room_type_id": str, "period": "YYYY-MM-DD", "revenue": str(Decimal), "count": int}]`

- `get_booking_trends(db: AsyncSession, start_date: date, end_date: date) -> list[dict]`
  - Query: `SELECT DATE(created_at) as day, COUNT(*) as count FROM bookings WHERE DATE(created_at) >= start_date AND DATE(created_at) <= end_date GROUP BY day ORDER BY day`
  - Return `[{"day": "YYYY-MM-DD", "value": int}]`

- `get_kpi_summary(db: AsyncSession, start_date: date, end_date: date) -> dict`
  - Query total_revenue: `SUM(total_price) WHERE check_in >= start AND check_in < end AND status IN ('confirmed', 'checked_in', 'checked_out')`
  - Query total_bookings: `COUNT(*) WHERE created_at >= start AND created_at <= end`
  - Query avg_daily_rate: `AVG(price_per_night) WHERE check_in >= start AND check_in < end AND status IN ('confirmed', 'checked_in', 'checked_out')`
  - Return `{"total_revenue": str, "total_bookings": int, "avg_daily_rate": str}`

**3. Room service occupancy endpoint** (`services/room/app/api/v1/reports.py`):

- Router prefix: `/api/v1/rooms/reports`
- Tags: `["room-reports"]`
- Import `require_staff` from room service deps (same pattern as booking service deps)

Endpoint:
- `GET /occupancy` — params: `start_date: date, end_date: date`
- Logic: Count rooms with each status (query Room table), plus query the reservation_projections table to count bookings overlapping each day in range.
  - A booking overlaps day D if check_in <= D AND check_out > D, status IN ('confirmed', 'checked_in', 'checked_out')
  - For ranges > 90 days: use a single efficient query approach. For shorter ranges: iterate days.
  - total_rooms: `SELECT COUNT(*) FROM rooms`
  - occupancy_pct per day = round(overlapping_count / total_rooms * 100, 1)
- Returns `{"daily": [{"day": "YYYY-MM-DD", "value": float}], "total_rooms": int, "avg_occupancy": float}`

Note: The room service has a `reservation_projections` table (from Phase 3) that tracks booking projections. Use it for occupancy counting.

- Returns merged JSON: `{"occupancy": {...}, "revenue": {...}, "trends": {...}, "kpis": {...}}`

- Timeout: 30.0 seconds (reports may be slower than operational endpoints)
