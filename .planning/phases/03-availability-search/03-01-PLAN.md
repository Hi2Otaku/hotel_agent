Build the reservation projection data layer, RabbitMQ event consumer, and availability query service for the Room service.

Purpose: This is the foundation for all search and availability features. The reservation projection table enables the Room service to answer availability queries without runtime calls to the Booking service. The event consumer keeps this table in sync via RabbitMQ. The availability service provides the core overlap-detection queries that search, detail, and calendar endpoints will call.

2. Create `services/room/app/services/availability.py`:
   - Import `uuid`, `date` from stdlib, `Decimal` from decimal
   - Import `select`, `func`, `and_` from sqlalchemy
   - Import `AsyncSession` from sqlalchemy.ext.asyncio
   - Import `ReservationProjection` from `app.models.reservation`

   - `BLOCKING_STATUSES = ["pending", "confirmed", "checked_in"]`

   - `def effective_capacity(physical_count: int, overbooking_pct: Decimal) -> int`:
     - `return int(physical_count * (1 + overbooking_pct / Decimal("100")))`

   - `async def get_available_count(db: AsyncSession, room_type_id: uuid.UUID, check_in: date, check_out: date) -> tuple[int, int]`:
     - Query 1: `SELECT count(*) FROM rooms WHERE room_type_id = :id AND is_active = true` -> `total`
     - Query 2: `SELECT count(*) FROM reservation_projections WHERE room_type_id = :id AND status IN BLOCKING_STATUSES AND check_in < :check_out AND check_out > :check_in` -> `blocking`
     - Query 3: `SELECT overbooking_pct FROM room_types WHERE id = :id` -> fetch overbooking_pct
     - Compute `capacity = effective_capacity(total, overbooking_pct)`
     - Return `(max(capacity - blocking, 0), total)`

   - `async def search_available_room_types(db: AsyncSession, check_in: date, check_out: date, guests: int, room_type_id: uuid.UUID | None = None, min_price: Decimal | None = None, max_price: Decimal | None = None, amenities: list[str] | None = None) -> list[dict]`:
     - Validate: check_in < check_out, guests >= 1
     - Query all active room types (filter by room_type_id if provided, filter by max_adults + max_children >= guests)
     - For each room type, call `get_available_count(db, rt.id, check_in, check_out)` -> (available, total)
     - Skip room types with available == 0
     - For each available room type, compute price using batch approach:
       - Call `get_base_rate_for_occupancy(db, rt.id, guests)` for price_per_night (use check_in date as representative)
       - Call `calculate_stay_price(db, rt.id, check_in, check_out, guests)` for total_price
     - Apply price filters (min_price, max_price against price_per_night)
     - Apply amenity filter: check if all requested amenities exist in rt.amenities values (flatten dict values to set)
     - Build result dict per room type with keys matching Plan 02 SearchResult schema exactly:
       `room_type_id`, `name`, `slug`, `description`, `photo_url` (first element of rt.photo_urls list, or None if empty), `price_per_night` (first night rate), `total_price`, `currency`, `max_adults`, `max_children`, `bed_config`, `amenity_highlights` (top 5 amenity keys from rt.amenities dict), `available_count`, `total_rooms`
       IMPORTANT: Use `photo_url` (singular, str|None) NOT `photo_urls` (list). Use `amenity_highlights` (list[str]) NOT `amenities`. These keys must match SearchResult(**r) unpacking in Plan 02.
     - Compute sort score using `compute_sort_score()` for default "recommended" sort
     - Sort results by score descending, return list

   - `def compute_sort_score(price_per_night: Decimal, max_price_in_results: Decimal, available_count: int, total_count: int, capacity_match: float) -> float`:
     - `price_score = 1.0 - float(price_per_night / max_price_in_results) if max_price_in_results > 0 else 0.5`
     - `avail_score = min(available_count / max(total_count, 1), 1.0)`
     - `cap_score = capacity_match`
     - `return (price_score * 0.4) + (avail_score * 0.3) + (cap_score * 0.3)`
     - `capacity_match` calculated as: `min(guests / max(rt.max_adults, 1), 1.0)` -- closer to full capacity = better match
