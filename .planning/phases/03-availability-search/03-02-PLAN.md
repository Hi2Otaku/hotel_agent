2. Create `services/room/app/api/v1/search.py`:

   ```python
   router = APIRouter(prefix="/api/v1/search", tags=["search"])
   # NOTE: No auth dependency on ANY endpoint in this file -- all public
   ```

   **Endpoint 1: GET /api/v1/search/availability**
   - Query params: check_in (date, required), check_out (date, required), guests (int, ge=1, required), room_type_id (UUID, optional), min_price (Decimal, optional), max_price (Decimal, optional), amenities (str, optional -- comma-separated), sort (str, default "recommended")
   - Depends on `get_db` only (from app.api.deps)
   - Validate: check_in < check_out (raise 400 if not), check_in >= today (raise 400 if in past)
   - Call `search_available_room_types(db, check_in, check_out, guests, room_type_id, min_price, max_price, amenities_list)`
   - Where `amenities_list = [a.strip() for a in amenities.split(",")] if amenities else None`
   - Return `SearchResponse(results=[SearchResult(**r) for r in results], total=len(results), check_in=check_in, check_out=check_out, guests=guests)`

   **Endpoint 2: GET /api/v1/search/room-types/{room_type_id}**
   - Query params: check_in (date, required), check_out (date, required), guests (int, ge=1, default=2)
   - Depends on `get_db` only
   - Fetch RoomType by id (raise 404 if not found or not active)
   - Call `get_available_count(db, room_type_id, check_in, check_out)` for availability
   - Call `calculate_stay_price(db, room_type_id, check_in, check_out, guests)` for pricing
   - Return `RoomTypeDetail` with all fields populated from RoomType model + availability + pricing

   **Endpoint 3: GET /api/v1/search/calendar**
   - Query params: room_type_id (UUID, optional), guests (int, ge=1, default=2), months (int, default=6, ge=1, le=12)
   - Depends on `get_db` only
   - If room_type_id provided, generate calendar for that type only. If not, use the cheapest active room type as default.
   - Generate dates from today to today + months using `timedelta(days=months * 30)` (avoid dateutil dependency)
   - Batch-load all rates for the room type: call `get_base_rates(db, rt_id)`, `get_seasonal_rates(db, rt_id)`, `get_weekend_surcharges(db, rt_id)` -- 3 queries total, NOT per-day
   - For each day, compute rate in-memory using multiplicative stacking (same formula as calculate_nightly_rate but without DB calls per day):
     - Find matching base rate for occupancy: `base_amount`
     - Find matching seasonal rate for date: `seasonal_mult` (or 1.00)
     - Check if weekday is in any weekend surcharge's days list: `weekend_mult` (or 1.00)
     - `final = (base_amount * seasonal_mult * weekend_mult).quantize(Decimal("0.01"))`
   - For each day, call `get_available_count(db, rt_id, day, day + timedelta(days=1))` for single-day availability
   - Availability indicator thresholds: green if available_count/total >= 0.5, yellow if >= 0.2, red if < 0.2 (or 0 available = red)
   - Return `CalendarResponse` with all days
