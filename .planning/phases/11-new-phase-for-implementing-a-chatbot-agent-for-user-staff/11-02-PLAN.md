    1. `services/chat/app/services/chat_engine.py`: Class `ChatEngine`:
       - Constructor: `__init__(self, db: AsyncSession, user: dict, llm: LLMProvider, tool_registry: ToolRegistry, tool_executor: ToolExecutor, prompt_builder: PromptBuilder, event_publisher)`
       - `async def process_message(self, request: SendMessageRequest) -> AsyncIterator[ServerSentEvent]`:
         a. If `request.conversation_id` is None, create new Conversation in DB, publish `chat.conversation.created` event
         b. If `request.confirmed_message_id` is set, load pending_confirmation from that message, execute the tool, yield ToolResultEvent, save result, then continue to LLM with tool result
         c. Load conversation history from DB (last 50 messages to stay within context window)
         d. Save user message to DB
         e. Build system prompt via `prompt_builder.build_system_prompt(conversation.bot_type, db)`
         f. Get tools via `tool_registry.get_tools(conversation.bot_type, user["role"])`
         g. **Tool-use loop** (max 5 iterations):
            - Call `llm.stream_message(messages, system, tools)`
            - For each StreamChunk:
              - `text_delta`: yield `ServerSentEvent(data=TextDeltaEvent(text=chunk.text).model_dump_json(), event="text_delta")`, accumulate text
              - `tool_use_start`: yield `ServerSentEvent(data=ToolStartEvent(tool_name=chunk.tool_name, tool_id=chunk.tool_id, description=_tool_description(chunk.tool_name, {})).model_dump_json(), event="tool_start")`
              - `tool_use_end`: Check `tool_registry.get_tool(chunk.tool_name)["requires_confirmation"]`:
                - If True: Save pending_confirmation to message, yield ConfirmationRequiredEvent, **return** (frontend will re-POST with confirmed_message_id)
                - If False: Execute tool via tool_executor, yield ToolResultEvent, append tool result to messages, **continue loop** (call LLM again with result)
              - `done`: break inner loop
            - If no tool calls in this iteration, break outer loop
         h. After stream completes:
            - Save assistant message to DB with accumulated content, token usage, estimated cost
            - Yield DoneEvent with message_id, token counts
            - Publish `chat.message.sent` event
            - If this is the first exchange (conversation has 2 messages), generate title asynchronously
         i. On error: retry once with 1s backoff, then yield ErrorEvent with `retryable=False`, message "The assistant is temporarily unavailable. Please try again in a moment."
       - Helper `_tool_description(tool_name: str, args: dict) -> str`: Maps tool names to human-readable descriptions per copywriting contract:
         - search_rooms: "Searching rooms for {check_in} to {check_out}..."
         - create_booking: "Creating your booking..."
         - cancel_booking: "Cancelling booking {confirmation_number}..."
         - check_in_guest: "Checking in {guest_name}..."
         - check_booking_status: "Checking booking status..."
         - etc.

    2. Update `services/chat/app/api/deps.py`: Implement rate_limiter:
       ```python
       RATE_LIMIT_MESSAGES = 20
       RATE_LIMIT_WINDOW = timedelta(minutes=1)

       async def rate_limiter(user=Depends(get_current_user), db=Depends(get_db)):
           since = datetime.utcnow() - RATE_LIMIT_WINDOW
           result = await db.execute(
               select(func.count(Message.id))
               .join(Conversation, Message.conversation_id == Conversation.id)
               .where(Conversation.user_id == uuid.UUID(user["id"]))
               .where(Message.role == "user")
               .where(Message.created_at >= since)
           )
           count = result.scalar() or 0
           if count >= RATE_LIMIT_MESSAGES:
               raise HTTPException(status_code=429, detail="Rate limit exceeded. Please wait before sending more messages.")
       ```
