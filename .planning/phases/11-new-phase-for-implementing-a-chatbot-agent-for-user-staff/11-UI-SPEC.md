# Phase 11 — UI Design Contract

> Visual and interaction contract for the Chatbot Agent (Guest + Staff). Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn (already initialized in both frontends) |
| Preset | new-york (slate base, CSS variables, no RSC) |
| Component library | Radix UI (via shadcn/ui) |
| Icon library | lucide-react |
| Font | Inter (sans-serif) via Google Fonts CDN (guest) / system stack (staff) |

**Existing shadcn components reused:** button, card, input, sheet, skeleton, avatar, badge, separator, dropdown-menu, dialog, sonner, tooltip, scroll-area

**New shadcn components to install:**
- `scroll-area` — chat message container with custom scrollbar (guest frontend; staff already has it via sidebar)

**Additional libraries:**
- `react-markdown` + `remark-gfm` — render markdown in bot message bubbles (bold, lists, code blocks, links)
- `eventsource-parser` — parse SSE streaming responses from chat service

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Inline icon gaps, timestamp padding |
| sm | 8px | Message bubble internal padding (horizontal), chip gaps |
| md | 16px | Message bubble vertical padding, sidebar item spacing |
| lg | 24px | Chat area horizontal padding, conversation list padding |
| xl | 32px | Page-level gutters |
| 2xl | 48px | Welcome message vertical spacing |

Exceptions: Message input area touch target minimum 44px height. Starter prompt chips minimum 36px height with 12px horizontal padding.

## Layout Contract

### Chat Page Structure (both frontends)

```
+------------------------------------------------------------------+
| Navbar (guest) / TopBar (staff)           [fixed, existing]      |
+------------------------------------------------------------------+
| Conversation   |  Chat Area                                      |
| Sidebar        |                                                  |
| (280px)        |  [Welcome / Messages]                           |
|                |                                                  |
| [New Chat]     |                                                  |
| conversation 1 |                                                  |
| conversation 2 |                                                  |
| conversation 3 |                                                  |
|                |  +------------------------------------------+   |
|                |  | Message input                    [Send]  |   |
|                |  +------------------------------------------+   |
+------------------------------------------------------------------+
```

### Chat Page Route

- Guest: `/chat` — rendered inside existing app shell (Navbar visible, no Footer)
- Staff: `/chat` — rendered inside existing AppLayout (Sidebar + TopBar visible)
- Both: full remaining viewport height below navbar (`h-[calc(100vh-64px)]`)

### Conversation Sidebar (left panel)

- Width: 280px on desktop
- Mobile: hidden, accessible via Sheet drawer (slide from left)
- Top: "New Chat" button (full width, accent outline variant)
- List: conversation titles sorted by most recent, each with date label
- Active conversation: accent left border (4px) + subtle accent background tint
- Right-click / long-press: context menu with "Rename" and "Delete" options
- Overflow: vertical scroll with scroll-area

### Chat Area (right panel)

- Flex column: messages container (flex-1 overflow-y-auto) + input bar (sticky bottom)
- Messages left-aligned (bot) and right-aligned (user)
- Max message width: 75% of chat area on desktop, 85% on mobile
- Auto-scroll to bottom on new messages; pause auto-scroll if user scrolls up
- Scroll-to-bottom button appears when not at bottom (floating, bottom-right of messages area)

### Message Bubbles

- Bot messages: left-aligned, avatar (32px) to the left, bubble with rounded corners (12px), max-width constraint
- User messages: right-aligned, no avatar, accent background, white text, rounded corners (12px)
- Rendered markdown inside bot bubbles (paragraphs, bold, italic, lists, code blocks, links)
- Timestamps: hidden by default, shown on hover/tap as tooltip below the bubble (12px, muted)

### Inline Room Cards (bot responses)

- Displayed within bot message flow as structured cards
- Photo thumbnail (80px x 60px, rounded-md), room type name (label weight 600), price per night (accent, semibold), top 3 amenities as small badges
- "Book this" action button (accent, small variant)
- Card: subtle border, rounded-lg, 16px padding

### Tool Action Status Cards

- Read actions (search, status check): green-tinted card with spinner then checkmark icon
- Copy pattern: "Searching rooms for Mar 25-28..." then updates to "Found 3 rooms" with results below
- Write actions (book, cancel, modify): amber-tinted card with confirmation prompt
- Confirmation prompt: action description + "Confirm" (accent) and "Cancel" (outline) buttons
- After confirmation: card updates to green-tinted success or red-tinted failure

### Starter Prompt Chips

- Displayed below welcome message in a centered flex-wrap container
- Each chip: rounded-full, border accent, text accent, 14px label, 12px horizontal padding, 36px height
- Hover: accent background with white text
- Guest chips: "Search rooms", "Check my booking", "Hotel info", "Cancellation policy"
- Staff chips: "Today's check-ins", "Room status", "Find guest", "Occupancy report"

### Welcome Message

- Centered in chat area when conversation is empty
- Bot avatar (48px) at top
- Heading: "Hi! I'm {bot name}" (20px semibold)
- Body: "I can help you with..." (16px regular, muted)
- Starter chips below (24px gap from body)
- Guest bot name: "HotelBook Assistant"
- Staff bot name: "HB Ops"

### Message Input Bar

- Sticky at bottom of chat area
- Full-width container with 16px horizontal padding, 12px vertical padding
- Input: multi-line textarea (auto-grow up to 4 lines, then scroll), rounded-lg, 16px body text
- Send button: icon-only (lucide Send icon), 40px square, accent background, rounded-full, right side of input
- Send on Enter (without Shift), newline on Shift+Enter
- Disabled state while bot is responding (input grayed, send button disabled)

### Loading / Typing Indicator

- Appears as a bot message bubble during response generation
- Animated three-dot bounce animation inside bot bubble shape
- Status text below dots: "Thinking..." or "Searching rooms..." (14px, muted, italic)

### Mobile Adaptations

- Conversation sidebar: hidden, triggered by hamburger/list icon in chat header
- Sidebar renders as Sheet (slide from left, 300ms ease-out)
- Message bubbles: max-width 85% (vs 75% desktop)
- Input bar: same layout, full width
- Inline room cards: stack photo above details (no side-by-side)
- Breakpoint: < 768px triggers mobile layout

## Interaction States

### Send Button
- Default: accent background, white send icon
- Hover: darken to `#0D6660` (guest) / lighten accent (staff)
- Disabled: 50% opacity (when input is empty or bot is responding)
- Active: scale 0.95

### Message Input
- Default: white background (guest) / card bg (staff), border
- Focus: accent border, ring shadow
- Disabled: muted background, "Waiting for response..." placeholder

### Conversation List Items
- Default: transparent background, label text
- Hover: subtle background tint
- Active: accent left border (4px), subtle accent background
- Context menu: dropdown with Rename and Delete options

### Starter Chips
- Default: border accent, text accent, transparent background
- Hover: accent background, white text, scale 1.02
- Active: scale 0.98

### Inline Action Buttons ("Book this", "Confirm", "Cancel")
- Follow existing button component interaction states
- "Book this": accent variant, small size
- "Confirm": accent variant, small size
- "Cancel": outline variant, small size

### Scroll-to-Bottom Button
- Floating circle (40px), bottom-right of messages area, 16px from edge
- Background: white (guest) / card bg (staff), shadow, border
- Icon: lucide ChevronDown
- Appears with fade-in (200ms) when user scrolls up
- Disappears when at bottom

---

## Loading States

### Chat Page Initial Load
- Conversation sidebar: 5 skeleton items (rectangle 280px x 48px each, pulse animation)
- Chat area: centered skeleton with avatar circle + 2 text lines (welcome message skeleton)

### Message Streaming
- Bot message bubble appears immediately with typing indicator (3 dots)
- Text streams in token-by-token (SSE), bubble grows as content arrives
- No skeleton for individual messages — typing indicator suffices

### Conversation History Load
- When switching conversations: 3 skeleton message pairs (bot + user) in chat area
- Fade out skeletons, fade in real messages (200ms)

### Tool Action In-Progress
- Status card with spinner icon (lucide Loader2, animate-spin) + action description text
- Updates in-place when action completes (spinner becomes checkmark or X)

## Accessibility Contract

- All interactive elements keyboard-navigable (tab order: sidebar -> chat messages -> input -> send)
- Focus ring visible on all focusable elements (accent color ring)
- Send button has aria-label "Send message"
- New Chat button is focusable and keyboard-activatable
- Message list uses role="log" with aria-live="polite" for new messages
- Bot typing indicator uses aria-live="polite" with sr-only text "Assistant is typing"
- Starter prompt chips are focusable buttons with descriptive labels
- Inline action buttons ("Book this", "Confirm", "Cancel") are accessible buttons
- Conversation sidebar list uses role="listbox" with aria-label "Conversations"
- Active conversation uses aria-selected="true"
- Delete confirmation dialog traps focus and is dismissible with Escape
- Scroll-to-bottom button has aria-label "Scroll to latest messages"
- Color contrast minimum 4.5:1 for all text (WCAG AA) — verified: white on teal-700 = 4.6:1
- Reduced motion: disable typing dot animation and chip hover scale via prefers-reduced-motion
