    2. `frontend/src/features/chat/api/chatApi.ts`: API functions using axios client:
       - `getConversations(): Promise<Conversation[]>` -- GET /api/v1/chat/conversations
       - `getMessages(conversationId: string, limit?: number, before?: string): Promise<ChatMessage[]>` -- GET /api/v1/chat/conversations/{id}/messages
       - `renameConversation(id: string, title: string): Promise<void>` -- PATCH /api/v1/chat/conversations/{id}

       - `sendMessageStream(content: string, botType: string, conversationId?: string, confirmedMessageId?: string): { response: Promise<Response>, abort: () => void }`:
         - Uses native `fetch` (not axios) for SSE streaming with POST
         - Creates AbortController, returns abort function for cleanup
         - Sets Authorization header from authStore token
         - Content-Type: application/json
         - Returns raw Response for eventsource-parser consumption

    4. `frontend/src/features/chat/hooks/useChat.ts`: Main chat hook:
       - Import `createParser` from `eventsource-parser`
       - `useChat()` returns `{ sendMessage, confirmAction, isStreaming, error }`
       - `sendMessage(content: string)`:
         a. Add user message to store
         b. Add empty assistant message with `isStreaming: true`
         c. Call `sendMessageStream` with content, "guest", currentConversationId
         d. Read response body as ReadableStream
         e. Feed chunks through eventsource-parser
         f. Handle events:
            - `text_delta`: appendToLastMessage
            - `tool_start`: addToolStatus with status "running"
            - `tool_result`: updateToolStatus with data and "success"/"error"
            - `confirmation_required`: setPendingConfirmation
            - `done`: setStreaming(false), update message id
            - `error`: show toast via sonner, setStreaming(false)
         g. On AbortController cleanup in useEffect return
       - `confirmAction(messageId: string)`:
         a. Call sendMessageStream with confirmed_message_id
         b. Same streaming handling
         c. Clear pendingConfirmation

    5. `frontend/src/features/chat/hooks/useConversations.ts`: React Query hook:
       - `useConversations()`: useQuery for getConversations, refetchInterval 30s
       - `useRenameConversation()`: useMutation for renameConversation, invalidates query
       - `useDeleteConversation()`: useMutation for deleteConversation, invalidates query
       - `useMessages(conversationId)`: useQuery for getMessages, enabled when conversationId is set

    6. `frontend/src/features/chat/hooks/useAutoScroll.ts`:
       - `useAutoScroll(ref: RefObject<HTMLElement>, deps: unknown[])`: Returns `{ isAtBottom, scrollToBottom }`
       - Tracks scroll position, auto-scrolls on new content if user hasn't scrolled up
       - `isAtBottom` is true when scrollTop + clientHeight >= scrollHeight - 50px threshold
       - `scrollToBottom()` scrolls to end with smooth behavior
